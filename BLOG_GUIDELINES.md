# LifeFlow 콘텐츠 가이드라인 (SSOT)

> 이 문서는 LifeFlow(life-revenue-blog) 모든 신규 글의 **작성 규격**이다. 수동 작성·n8n 자동생성 모두 이걸 따른다.
> 목적: 기존 글의 "총정리·완벽가이드" 단조 틀을 버리고 **후킹 + 전환 유도**형 구조로 개편.

---

## 0. 병 진단 (왜 개편하나)

기존 LF 글은 제목이 전부 `~2026 / ~총정리 / ~완벽 가이드 / top5` 한 틀이고, 본문은 "왜 놓치기 쉬울까 → 표 → 일반론 → 신중히 확인하세요"의 반복이다. **관점(POV)도, 나만 아는 구체 수치도 없어** 다른 블로그 1천 개와 구분이 안 된다(AI 양산형).

⚠️ **전제**: 톤 개편은 B급 레버다. LF 검색 사망의 A급 병목은 여전히 ①자체 도메인(vercel.app 네이버 천장) ②Threads 유통이다. **단, 발행하는 글은 반드시 아래 공식을 따른다.**

---

## 1. 목소리 3원칙 (단조로움 킬러)

1. **관점(POV) 1문장** — "다들 A라는데 사실 B" 같은 각도를 서두에 못 박는다. 중립 나열 금지.
2. **구체성** — 일반론("소득 기준이 복잡합니다") 대신 숫자·조건·상황("연봉 3,600만 원 직장인 기준, 컷오프는 중위소득 150%").
3. **정체성 훅** — "누구를 위한 글"인지 명시. 전체 대상 = 아무 대상.

**말투**: 블로그 본문은 **존댓말(~습니다/~하세요)** 유지 (반말·스친체는 Threads 전용). 단 서두는 뻔하지 않게.

**⛔ 안티패턴 (하지 마라)**
- 지어낸 개인 경험·영수증("제가 받아봤는데") — AI 블로그가 지어내면 신뢰(E-E-A-T)·정책 리스크. 숫자는 실데이터 출처로만.
- 볼드 남용(문장마다 `**`) — AI 티. 핵심 수치 몇 곳만 강조.
- disclaimer 남발("반드시 확인하세요"를 문단마다). 경고는 글당 1~2개면 충분.
- "매년 하반기가 되면…", "~는 많은 분들이 궁금해하는 주제입니다" 같은 뻔한 서두.
- 제목에 `총정리·완벽가이드·top5` (구체성 없는 라벨).

## 2. 제목 공식 5종 (택 1, ≤ 32자 권장)

| # | 유형 | 틀 | 예시 |
|---|------|----|------|
| 1 | 구체 숫자/영수증 | [대상]이 [행동]하면 [구체 결과·금액] | 연봉 3,600 직장인이 놓치는 지원금 5개, 합산 340만 원 |
| 2 | 역발상/통념반박 | [통념]은 사실 [반박] | 여름 다이어트 실패는 의지 문제가 아니다 |
| 3 | 정체성 타겟 | [구체 대상]을 위한 [약속] | 혼자 사는 사회초년생의 여름 전기세 4만 원 셋업 |
| 4 | 손실회피/경고 | [행동] 안 하면 [손실] | 7월 안에 안 챙기면 사라지는 정부지원금 3가지 |
| 5 | 질문/호기심갭 | [의외의 질문]? | 예금 금리 3%인데 왜 손에 쥐는 건 2%일까 |

규칙: 숫자는 구체값(‘여러 개’ ❌ → ‘5개’ ⭕). 낚시 과장 금지(정책·신뢰 리스크).

## 3. 인트로 공식 (첫 3문장 = 이탈 방지 구간)

H1 바로 다음, 3문장으로:
1. **훅** — 문제·정체성·약속 중 하나로 시작 (뻔한 서두 금지).
2. **공감/구체화** — 독자의 실제 상황을 한 컷 그린다.
3. **약속 + 스크롤 유도** — "이 글에서 [구체적으로 얻을 것]을 정리했습니다."

예)
> 정부지원금은 몰라서 못 받는 게 대부분입니다. 연봉이 3천만 원대면 "나는 대상 아니겠지" 하고 넘기지만, 실제론 3~5개가 걸립니다. 이 글은 직장인이 자주 놓치는 지원금을, 소득 구간과 신청처까지 붙여 정리했습니다.

## 4. 포스트 골격 (전환 최적화 구조) — 이 순서를 기본형으로

```
1. # H1 (제목 공식)
2. 인트로 훅 (3문장, §3)
3. 📌 핵심 요약  ← callout-info 로 TL;DR 3~4줄 (체류·스크롤 유도)
4. [Problem] 왜 이게 문제인가 / 왜 놓치나  (공감·긴장 조성)
5. [본론] H2 섹션 2~4개
     · 각 섹션에 표 또는 chart-bar 1개 (근거 = 체류시간·신뢰)
     · 제품 맥락이 자연스러운 지점에 coupang-inline 1개 (§5)
6. [실전] 체크리스트 / 단계표  ← chart-progress 또는 표 (행동 유도)
7. [흔한 실수]  ← callout-warning 1개
8. [더 읽기] 본문 내부링크 2~3개 유도 문단 (§5)
9. [마무리] 댓글 유도 질문 1개 (§5, CommentSection 활성화)
10. FAQ  ← frontmatter faq 4~5개
```

레이아웃이 본문 뒤에 **자동 주입**: `공유 버튼 → RelatedPosts → NewsletterCTA → CommentSection`. 그러니 본문 마지막은 **댓글 질문**으로 끝내 자연스럽게 연결한다.

## 5. 전환 유도 규칙 (유도 많이 되게)

- **쿠팡 (coupang-inline)**: 글당 **1~2개**, "제품이 실제로 필요한 맥락"에서만(억지 삽입 = 신뢰 하락). frontmatter `coupangLinks` 2개 + 본문 `coupang-inline` 삽입. lptag=AF7838146 정적 검색 URL만(딥링크/API 금지 → 계정제한).
- **내부링크**: 본문에 자연 앵커 **2~3개**(관련 LF 글) + 하단 RelatedPosts 자동. 체류·PV 복리.
- **댓글 유도**: 마무리에 **독자 겨냥 구체 질문 1개** — "여러분은 이번 여름 전기세 얼마 나왔나요?" 식. (일반적 "댓글 남겨주세요" ❌)
- **callout 역할 분리**: `info`=요약/정보, `warning`=실수/마감경고, `tip`=꿀팁. 남발 금지(글당 각 1~2개).
- **차트**: `chart-bar`=비교·랭킹·구간, `chart-progress`=단계·체크·달성률. 데이터에 유형 매칭.
- **광고(AdSlot)**: 자동. 본문 길이(9,000자+)·소제목·시각요소로 **스크롤을 늘리는 것**이 광고 노출·수익의 간접 유도.

## 6. 컴포넌트 스니펫 (복붙용)

**Callout 3종**
```html
<div class="callout-info">📌 핵심 요약: …</div>
<div class="callout-warning">⚠️ 주의: …</div>
<div class="callout-tip">💡 팁: …</div>
```

**쿠팡 인라인** (텍스트만 교체, 구조 유지)
```html
<a class="coupang-inline" href="{lptag=AF7838146 정적 검색 URL}" target="_blank" rel="noopener noreferrer nofollow">
  <span class="ci-icon" aria-hidden="true">📁</span>
  <span class="ci-body">
    <span class="ci-label">쿠팡 추천</span>
    <span class="ci-title">{맥락에 맞는 제품 한 줄}</span>
  </span>
  <span class="ci-cta">최저가 →</span>
</a>
```

**차트** (⚠️ 속성은 **복수형** `data-labels`/`data-values`. 한 div = 여러 막대 = 차트 하나. `data-label`/`data-value` 단수 쓰면 값이 안 읽혀 **전부 0으로 렌더**됨. ⚠️ data-labels 안에 천단위 쉼표 금지 → 라벨 쪼개짐)
```html
<!-- 비교·랭킹·구간 (수평 막대) -->
<div class="chart-bar" data-labels="예금,적금,파킹" data-values="3.1,3.4,2.8" data-unit="%"></div>
<!-- 짧은 라벨(8자↓·2~6개) 비교는 세로 막대. 제목은 결론형, 핵심 하나는 data-highlight로 강조 -->
<div class="chart-bar" data-orient="vertical" data-highlight="max" data-title="적금 금리가 가장 높다" data-labels="예금,적금,파킹" data-values="3.1,3.4,2.8" data-unit="%"></div>
<!-- 단계·구성·달성률 (진행 막대, data-max 기준 %) -->
<div class="chart-progress" data-title="표시 이자 100 중 남는 비율" data-labels="세전,세후,실질" data-values="100,84.6,18" data-max="100" data-unit="%"></div>
```
공통 옵션: `data-title`, `data-colors`(쉼표 병렬), `data-max`, `data-unit`. ⚠️ **단계별 계산·공식은 차트가 아니라 번호 목록/표로** — 진행바는 '값의 크기 비교'용이지 '순서 나열'용이 아니다.

## 7. 프론트매터 체크리스트 (LF 형식 — 틀리면 OG/색인 깨짐)

```yaml
title: "…"                 # 제목 공식 §2, ≤32자
description: "…"           # 검색 스니펫, 150자 내, 훅 포함
pubDate: "2026-07-16"      # ⚠️ 문자열(따옴표)
author: "LifeFlow"
category: finance           # ⚠️ 소문자: finance/education/health/lifestyle/travel
tags: [태그1, 태그2]         # ⚠️ 슬래시(/) 금지 → 빌드 깨짐
heroImage: "https://images.pexels.com/…w=1200"   # ⚠️ 플랫 문자열=og:image, 누락 시 소셜카드 깨짐
coupangLinks: [ {title, url}, {title, url} ]      # lptag=AF7838146 정적 검색 URL
faq: [ {q, a}, … ]         # 4~5개, 각 3~4문장
```
상세: 레포별 관례 차이 → 메모 `project_blog_frontmatter_conventions`.
